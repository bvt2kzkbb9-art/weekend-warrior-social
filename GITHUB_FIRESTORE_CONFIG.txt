✅ KROK 1: Firebase CLI Setup
────────────────────────────
Sprawdź czy Firebase CLI zainstalowany:
firebase --version

Powinna być: firebase-tools/15.x.x lub wyżej

Jeśli nie zainstalowany:
npm install -g firebase-tools

✅ KROK 2: Zalogowanie do Firebase
───────────────────────────────────
firebase login

To otworzy przeglądarkę:
1. Zaloguj się na konto Google/GitHub z dostępem do Firebase
2. Zezwól Firebase CLI dostęp do projektu
3. Wróć do terminala - powinno się zalogować

Sprawdzenie:
firebase projects:list

Powinieneś zobaczyć:
weekend-warrior-social-ed3d0

✅ KROK 3: Ustaw bieżący projekt
────────────────────────────────
firebase use weekend-warrior-social-ed3d0

Potwierdzenie:
firebase projects:describe weekend-warrior-social-ed3d0

✅ KROK 4: Firestore Database Setup
────────────────────────────────────
Przejdź do: https://console.firebase.google.com/project/weekend-warrior-social-ed3d0

Menu:
1. Firestore Database
2. Kliknij "Create Database"
3. Modus: Uruchom w trybie testowania (do testów) lub produkcji (ze rules)
4. Lokalizacja: us-central1 (lub bliżej Ciebie)
5. Kliknij "Enable"

UWAGA: Baza danych powinna być już utworzona. Jeśli pytanie pojawia się,
oznacza to że trzeba ją włączyć w Firebase Console.

✅ KROK 5: Firestore Security Rules - Deploy
──────────────────────────────────────────────
Sprawdzenie reguł:
cat firestore.rules | head -20

Deploy reguł:
firebase deploy --only firestore:rules

Powinno być:
✔ Deploy complete!

Weryfikacja w konsoli:
https://console.firebase.google.com/project/weekend-warrior-social-ed3d0/firestore/rules

✅ KROK 6: Cloud Storage Rules - Deploy
────────────────────────────────────────
Sprawdzenie reguł:
cat storage.rules

firebase deploy --only storage

Powinno być:
✔ Deploy complete!

Weryfikacja w konsoli:

✅ KROK 8: Firebase Authentication - Setup
───────────────────────────────────────────
Przejdź do:
https://console.firebase.google.com/project/weekend-warrior-social-ed3d0/authentication

   - Kliknij "Email/Password"

CZĘŚĆ 3: DEPLOYMENT - OSTATECZNY KROK
═══════════════════════════════════════════════════════════════════════════════

Upewnij się że:
✅ Zalogowany w Firebase (firebase login)
✅ Projekt ustawiony (firebase use weekend-warrior-social-ed3d0)
✅ Branch poprawny (git branch --show-current)
✅ Pliki są na miejscu (ls firebase.json firestore.rules storage.rules)

Uruchom deployment:
───────────────────
cd /home/user/weekend-warrior-social

./deploy.sh

LUB ręcznie:

firebase deploy --only firestore:rules
firebase deploy --only storage
firebase deploy --only firestore:indexes
firebase deploy --only hosting

⏱️  To potrwa 20-30 minut!

Czekaj na:
✔ Deploy complete!

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

❌ "Permission denied" przy firebase login

✓ firebase login (wpisz ponownie)

❌ "Project not found" przy firebase use
✓ Sprawdź czy zalogowałeś się (firebase projects:list)
✓ Wpisz ID: weekend-warrior-social-ed3d0 (bez zmian)

❌ "Firestore Database not found"
✓ Przejdź do Firebase Console
✓ Kliknij "Firestore Database"
✓ Kliknij "Create Database"
✓ Wybierz tryb i lokalizację
✓ Kliknij "Enable"

❌ "Storage bucket not found"
✓ Przejdź do Firebase Console
✓ Storage > Create bucket
✓ Użyj domyślnej nazwy

❌ "Hosting returns 404"
✓ ./deploy.sh --only hosting
✓ Czekaj 2-5 minut
✓ Czyszcz cache przeglądarki (Ctrl+Shift+Delete)

❌ "Rules deployment failed"
✓ Sprawdzenie syntaxu: firebase validate
✓ Spróbuj ponownie: firebase deploy --only firestore:rules
